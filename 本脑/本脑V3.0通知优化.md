# 本脑V3.0版本通知优化

标签（空格分隔）： 360BrainSecurity

---

## 通知方式
- 邮件：需配置邮件主题、邮件内容，收件人、抄送人(选填)
- 阿里云短信：收件人
- 钉钉：收件人，URL
- 企业微信：收件人
- webhook：url
其中，webhook不需要构造内容。目前的策略是邮件和其他通知方式模板有所区别，邮件通过引入html模板，通过模板构造发送文本来发送，其他通知方式的文字描述都有字数限制，只需要构造一段能描述具体信息的文本即可，模板暂定一致。
<font color="Red">**本脑V3.5版本增加织语通知方式**</font>

## 通知方式配置整理

  | 通知方式       | system_config type  | 配置内容  |
  | ---------- | ------ | ------ |
  | 邮件 | 	SMTP | SMTP服务器地址、端口，用户名密码 |
  | 阿里云短信 | 	SMS_CLOUD | 用户阿里云短信id、密钥，短信签名，用户还需配置模板id |
  | 钉钉 | 	DINGTALK | 收件人、url |
  | 企业微信 | 	WECHAT | agentID、corpId，密钥 |
 
## 支持通知的模块
- 原始告警(sae/ice): 设置发送频率
- 安全事件(ice): 安全事件创建、处置时发送通知，处置及变更责任人时给对应owner发送邮件通知
- 系统监控(monitor): 监控出现异常时发送通知
- 报表(reporter): 新建报表任务时发送通知
- 数据接入(dv): 针对每个数据源，对其采集监控做配置，主要包括解析规则慢告警、解析字段数异常告警、核心字段解析异常告警、解析失败告警、数据量异常告警、日志发生时间和接收时间差值异常告警。通知频率为半小时一次。数据接入页面，打开通知配置开关，然后进入采集监控页面，选择右上角的触发条件，满足就会触发通知。
- 工单(work_order): 创建、工单状态流转时发送邮件通知
- 指挥调度(tresdin): 催办时发送邮件通知

## 阿里云短信通知模板
短信模板规范：https://help.aliyun.com/document_detail/463161.html
不支持ip
模板：
```
告警
你好，本地安全大脑平台于${start_time}检测到告警: ${alarm_name}，请及时关注。

安全事件
创建：你好，本地安全大脑平台于${start_time}生成一例安全事件: ${title}，请及时关注。

系统监控
${start_time}，模块${module_name}发出警告: ${alarm_content}，${alarm_advice}。

报表
${type}类报表${report_title}于${generate_time}生成，请及时查看。

数据接入
采集器数据源${collector_source}于${current_time}发出监控预警，请及时关注。
```

## 各种通知方式的内容模板

### **原始告警**
主题：[@{alarm_level}]告警通知: @{alarm_name}
内容：
```
告警名称: @{alarm_name}
开始时间: @{start_time}
更新时间: @{end_time}
攻击场景: @{sae_rule_type}
攻击者: @{attacker_array}
受害者: @{victim_array}
告警级别: @{alarm_level}
告警阶段: @{alarm_stage}
告警内容: @{alarm_content}
处置建议: @{alarm_advice}
```
