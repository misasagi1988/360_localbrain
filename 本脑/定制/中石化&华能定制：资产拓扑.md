
## 需求说明

|   |   |   |
|---|---|---|
|**需求阶段**|**需求描述**|**需求来源**|
|第一阶段|拓扑管理：支持拖拽式绘制拓扑，并对创建的拓扑进行统一管理，支持基本的增删改查操作|中石化、华能澜沧|
|分权分域：总部支持查看全部，下级租户相互隔离|中石化|
|第二阶段|拓扑发布：支持将拓扑发布，显示拓扑的风险区域（安全域：根据告警、脆弱性、资产重要程度计算出区域风险再展示）、风险设备（体现攻击告警的总数等），支持查看富化信息、下钻等操作|中石化|
|拓扑发布到大屏|本脑原有功能|

### 拓扑画布组件区域说明

提供四类基础的组件，组件支持自由拖拽到画布中进行绘制
i. 通用组件：文字、矩形、直线；
ii. 安全设备：以图标+文字形式，每一个图标代表一种安全设备类型，图标包括：通用安全设备、防火墙、Web应用防火墙（WAF）、下一代防火墙（NGFW）、堡垒机、VPN、主机入侵检测系统（HIDS）、入侵检测系统(IDS)、入侵防御系统(IPS)、终端保护平台(EPP)、终端威胁检测及响应(EDR)、网络威胁检测及响应(NDR)、沙箱、蜜罐、云工作保护平台（CWPP）、云WAF、运行时应用自我保护平台(RASP)、邮件网关(EmailGW)、数据泄露保护(DLP)；
iii. 安全域；
iv. 网络设备：路由器、交换机、负载均衡、主机、服务器、数据库、网络、云；

其中，安全设备、安全域、网络设备需要额外添加一些其他配置。不同的组件类型对应有不同的配置页面：
i. 安全设备/安全域通过关联已管理的安全设备/安全域进行配置，关联后展示对应信息；（4.0定制版本，安全设备为引用，安全域为配置名称）
ii. 矩形支持配置下钻的拓扑，完成拓扑之间的跳转；
iii. 路由器/交换机/负载均衡需配置设备名称和设备IP；
iv. 网络/云需配置设备名称、描述、地址范围（支持关联网段、手动添加，支持IPV6格式）；
v. 主机/服务器/数据库配置设备名称、描述、关联资产（上限1000个，超过时提示“已超出关联资产数量上限，无法再继续关联资产”）。

### 发布/投屏分权

在开启分权状态下，仅总部组织机构存在发布/投屏操作权限，发布后的拓扑所有组织机构用户均可查看，仅对拓扑节点的详情内容进行分权：
- 资产：点击详情后需根据所属组织机构过滤，仅支持查看当级及其下级组织机构的资产信息，若不存在则显示“暂无数据”；
- 安全设备：若无查看权限，则提示“该安全设备属于xxx组织机构，暂无查看权限。”
- 安全域：直接根据分权过滤无数据/无权限的资产/安全设备；
- 所有角标数字、风险等级均需根据分权过滤后的数值进行统计；（大屏同理）
未开启分权状态下，所有组织机构都支持进行发布/投屏操作，且开放所有数据的查看权限。

### 拓扑发布展示内容说明

显示内容包括拓扑名称、统计时间、拓扑图（配置的网络拓扑、风险等级、告警数量等，告警数量分为两类，安全设备右上角为上报告警数量，资产右上角为遭受攻击的数量）。

**安全设备：**显示安全设备上报告警数量。点击后展示安全设备的详细信息，包括基础信息、风险信息两大部分：
1.      基础信息：展示设备详细配置信息，并支持跳转到【**资漏中心-安全设备管理**】页面对安全设备进行管理；
2.      风险信息：包括上报告警数量统计（总数、各等级数量）、告警趋势（受控于时间组件）、近期上报告警列表（展示最近5条告警），点击【更多】跳转到对应的页面查看全部信息；什么页面？

**资产节点：**显示资产告警数。点击后展示资产节点的详细信息，包括基础信息、资产信息、风险信息三大部分：这里仅指主机/服务器/数据库？

1.      基础信息：包括设备名称、描述；
2.      资产信息：展示资产统计值（总数、重要资产数）和资产列表；
3.      风险信息：包括风险信息统计值（告警总数、漏洞数、风险资产）和对应的风险列表，风险列表默认展示近期告警（最近5条），支持选择tab页进行切换，点击【更多】跳转到对应的页面查看全部信息；什么页面？这里不需要展示风险等级？
4.      支持通过输入IP/IP段对资产信息和风险信息模块进行筛选。

**安全域：**拓扑中需显示安全域的风险等级，通过计算该安全域内所有资产风险值后综合得出。点击后以抽屉形式展示安全域的详细信息，包括基础信息、资产信息、风险信息三大部分：

1.      基础信息：展示安全域详细配置信息，并支持跳转到【**评估中心-评估环境-安全域**】页面对安全域进行管理；
2.      资产信息：包括该安全域下的资产信息统计值（资产总数、重要资产）和资产列表；如何定义安全域下的资产？如何定义安全域下的告警？如何定义安全域下的漏洞？
3.      风险信息：包括风险信息统计值（告警总数、漏洞数、风险资产）和对应的风险列表，风险列表默认展示近期告警（最近5条），支持选择tab页进行切换，点击【更多】跳转到对应的页面查看全部信息；
4.      支持通过输入IP/IP段对安全域内的资产信息和风险信息模块进行筛选。

## 功能设计思路

### 已知信息

1. 拓扑图支持级联嵌套
2. 导出拓扑时，仅保留节点和连线，节点的所有配置信息全部删除
3. 拓扑跟组织机构走，拓扑属于创建用户的组织机构，非该组织机构的任何用户对该拓扑无编辑和删除权限
4. 组织机构删除时，相关拓扑的所属组织机构改为其父节点
5. 删除拓扑时，如果该拓扑被其他拓扑引用，需给出提示，给前端的接口中，尽量说明引用该拓扑的父拓扑信息，供前端判断
6. 拓扑一次仅支持发布一个，投屏一个
7. 资产ip不支持更新
8. 只有矩形框和安全域可以支持包含其他组件。矩形框支持包含所有其他组件，安全域不支持嵌套安全域，其他基础组件都支持。
9. 在实际计算告警、风险等级一类的值时，**==无需关注其页面配置的嵌套关系==**，仅关注组件实际配置的数据即可。
10. 安全设备关联上报告警: 安全设备有ip地址，dev_ip_list里包含该ip的告警为该安全设备关联的告警
11. 网络设备(资产)关联告警: 网络设备关联资产，victim_array里包含该ip的告警为该网络设备关联的告警
12. 安全域关联告警: 拓扑图中安全域包含的网络设备关联告警，无需计算嵌套子节点的。
13. 安全域安全等级计算方法: 通过计算拓扑图中安全域包含的网络设备资产的安全等级得到，无需计算嵌套子节点的。

需要记录拓扑引用的子拓扑信息，删除时给出提示。
导出时直接将组件data值为null，移除所有业务信息。

### update, 2024-10-9，讨论后更新设计思路

之前一直以为自己需要记录并解析拓扑图，不同节点的配置自己需要做解析，展示详情时自己做过滤。但他们的意思是这部分完全不用，只需要前端自己去解析。如果后端有需要关注的部分，只需要数据表另外加一个字段，前端提供，后端保存即可。
讨论后决定：
1. 对于拓扑结构，后端只需要保存即可，无需了解拓扑图各个节点的配置详情；尤其针对保存的资产、网段信息，后端无需关注，前端在拿到配置后，完全可以根据配置，另外调用其他接口查询关联内容。关联资产信息，建议保存ip；关联网段信息，建议保存id
2. 拓扑图中安全域风险等级: 通过计算该安全域内所有资产风险值后综合得出，后端提供接口，前端请求中指明该安全域包含的所有资产信息，后端计算后返回结果；
3. 安全域节点详细信息-资产概览: ~~后端提供接口，前端请求中指定该安全域包含的所有网络资产设备，各个资产设备包含的资产，后端据此对各个设备的资产总数、重要资产数、告警总数、漏洞数、风险资产数做统计后返回给前端~~。后端提供接口，前端请求中，给出网络资产设备id，设备包含的资产ip列表，后端据此对该设备关联的资产总数、重要资产数、告警总数、漏洞数、风险资产数做统计后返回给前端。
4. 拓扑图中安全设备上报告警量: 安全设备根据ip与告警做关联，直接复用本脑当前已有接口即可实现数据统计，无需额外提供接口；
5. 安全设备节点详情-风险信息: 包括上报告警数量统计（总数、各等级数量）、告警趋势（受控于时间组件）、近期上报告警列表（展示最近5条告警），直接复用本脑当前已有接口即可。
6. 拓扑图中网络设备(资产)关联告警数: 类似安全域节点资产概览部分，确认下只提供一个接口供前端使用。
7. 资产节点详情-资产信息及风险信息: 支持通过输入IP/IP段对资产信息和风险信息模块进行筛选，确认下是否直接复用本脑当前已有接口即可。

## 华能交付后，测试提出的几个问题的讨论及改动(2024/12/9)
1. 无关联实体及资产，相关告警被搜出  
原因: 我们实体的抽取逻辑是写死的，只从日志中的固定字段中提取，而告警受害者组的逻辑是在威胁分析规则中任意可配置的，这两边一定存在对不齐的情况。  
解决方案: 数据造假，无关联实体及资产，关联告警数量强制置0
2. 数据分权相关：分权情况下，下级组织机构无法看到关联实体IP、资产，但会查到告警。  
原因: 告警的分权逻辑，组织机构保存有用户所属组织机构信息，即可被查出。
解决方案: 节点引用的关联网段信息，不同用户登录时，也做分权查询，不同用户能看到的节点引用的关联网段不同，告警、实体、脆弱性等查询使用的网段也不同。同时，保持上面逻辑：数据造假，无关联实体及资产，关联告警数量强制置0
