
### 隔离域

只有两个隔离域，即
- 总部
- 租户（租户A，租户B，...）

### 如何获取用户所属租户信息

前端请求header中增加Tenant_id_path项，后端可以优先从header中获取租户信息。

### 影响范围

- dv: 数据采集测必须为每条接入的日志富化出`租户ID`字段
- 威胁分析规则、调整规则、合并规则和加白规则: 根据租户隔离，可见性、过滤条件按租户id对数据做过滤补充
- 安全信息: 信息组对所有人都是可见可引用的，内置的信息组只有总部租户可编辑，其他总部或租户自建的信息组只有租户自己可以编辑；导入相关：信息组导出时不带租户信息，导入时，如果信息组存在，租户需确认该信息组自己有无权限编辑(信息组租户id与自身是否一致)

### 安全信息改造

 "security_intelligence_group"数据表增加"tenant_id_path"字段
 ```
ALTER TABLE security_intelligence_group
ADD tenant_id_path varchar(255) DEFAULT NULL COMMENT '用户所属租户ID路径';
```
