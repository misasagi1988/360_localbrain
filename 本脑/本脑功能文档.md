## 各种属性类别含义

关联分析属性：只有关联分析规则使用到的属性，dv不能使用的
基础属性：安全和用户定义的属性
系统属性：解析规则不可解析的字段，一般都是解析规则自己使用的
业务属性：产品定义的安全可以使用，但是不能更改的字段，
高级业务属性：情报，资产，soar，ueba等使用到的字段

## 本脑性能监控
这个需求，主要是现在的客户，前端使用人员越来越多，像农行这样，一个总行，36加分行，公用一套系统，前端查询也没太多限制，经常容易出现一些问题，尤其是在护网期间，希望通过这些监控，能帮助我们快速发现和定位问题，定位方法能自动化的，能图表展示的，就逐渐的建设起来，减少支持和维护成本。
1、背景流量监控
   --dcc、sae、ice、shuri、artifact各个点的处理速度，读写、生成告警、生成事件等
   --其他必要仪表盘

sae: 加载规则量、规则耗时，规则状态，自定义函数性能，热点规则，消费速度，缓存队列，kafka lag
是否要定义一个性能阈值，lower limit，upper limit？
性能监控是基于规则的，还是全局的？需要有权限的？
ice、sae告警生成、告警入库延迟问题排查方法以及相关排查方法自动化
ice、sae排查问题需要的相关监控
引擎状态监控: /api/cep/core/debug/metric/engine，engine metric 1min一次计入，最多记录100条数据
规则状态监控: api/cep/core/debug/metric/statement，规则耗时统计，无排序，规则trigger即记录。最多记录100条数据
规则耗时统计: /api/cep/core/debug/metric/statement/period，规则耗时统计，将cache中的多个规则耗时数据按cpuTime排序，最多记录10条数据。1s清理一次cache
UDF性能概要: /api/cep/core/debug/metric/udf
规则状态大小统计: /api/cep/core/debug/metric/statement/memory，状态数据不清。

| state        | 所含内容                   | 描述                       |
| ----------- | ------------- | ------------------------- |
| WinStateMetric   |name: 规则名称<br/>	innerWinSize: 仍在时间窗口内的事件数量 | 无分组，count模板|
| GroupWinStateMetric   |name: 规则名称<br/>		groupSize: 仍在时间窗口内的分组数量<br/>	innerWinSize: 仍在时间窗口内的事件数量<br/>	groupWinSize: 仍在时间窗口内的事件数量 |有分组，count模板 |
| DistinctWinStateMetric   |name: 规则名称<br/>	distinctWinSize: 共享这些distinct字段值的事件数量<br/>	innerWinSize: 仍在时间窗口内的事件数量 |无分组，distinct模板 |
| DistinctGroupWinStateMetric   |name: 规则名称<br/>		groupSize<br/>	innerWinSize<br/>	groupWinSize |有分组，distinct模板 |
| RlikeStateMetric   |name: 正则<br/>	counter:正则使用计数 | 正则表达式|
| PatternStateMetric   |name: 规则名称<br/>	subExpression：规则subExpression数量 | 关联类模板|
