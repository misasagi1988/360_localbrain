### 详细设计

#### DB配置

新增 **自定义智能体研判配置数据表**，gpt_agent_config, 至少需要包括如下关键字段

| 字段        | 类型           | 功能描述             |
| -------------- | --------------------- | ------------------ |
| id          | string         | id           |
| app_id          | string         | 智能体id           |
| app_name      | string  | 智能体名称           |
| desc          | string                | 智能体描述        |
| hql         | string         | 智能体研判范围过滤条件           |
| epl         | string         | 智能体研判范围过滤条件语法转换           |
| status         | int         | 智能体状态(1值守中/0待命中)        |
| publish_status         | int         | 智能体发布状态，1发布，0未发布，2已删除        |
| priority         | int         | 智能体优先级       |
| system         | int         | 智能体是否内置       |
| creator_id         | string         | 创建用户id           |
| updater_id         | string         | 更新用户id           |
| create_time         | timestamp         | 智能体添加时间      |

#### API接口设计

查看智能体列表，智能体名称模糊搜索，仅看值守中筛选。是否支持分页展示？
支持分页，排序按添加时间排序。智能体图标万事通传给我们；
- 查询接口
```
get: /gpt/agent
查询参数:
page: 分页
page_size: 分页大小
name: 智能体专家名称，模糊查询
status: 仅看值守中筛选
response body:
{
	"data": {
		"total": 3,   //总数
		"page": 1,   //当前页
		"list": [{
			"id": "131E7QH20000",   //配置id
			"appId": "c9b90f0ff3c6496e8387a56564a7e122",   //智能体id
			"appName": "告警运营专家",   //智能体名称
			"desc": "desc",   //智能体描述
			"status": 1,   //启停状态，0表示停止
			"system": 1,   //是否内置，1表示是
			"priority": 3,   //优先级，值越小优先级越高
			"hql": {    //研判范围配置，与本脑gpt sp3告警智能研判配置数据格式相同
  			        "autoOpeSwitch": true,   //自动运营开关，true
  			        "deepJudgement": true,  //深度研判开关
  			        "deepJudgementHql": "安全大模型研判 = \"攻击成功\" and 告警类型 != \"威胁情报\"",  //深度研判 
  			        "deepPreFilterHql": "安全大模型研判 exist",
  			        "multipleJudgment": true,
  			        "noiseReduceHql": "告警级别 = \"严重\"",   //快速降噪
  			        "noiseReduceReset": false,
  			        "deepJudgementReset": false,
  			        "similarWarnInit": false
			},
			"lastGptResultTime": 1774595529245,   //最近一次研判时间
			"gptResultCount": 100,   //近24h研判告警数
			"createTime": "2026-01-23 18:36:30",    //添加时间
			"updateTime": "2026-01-23 18:36:30"    //更新时间
		},...]
	},
	"messages": [],
	"statusCode": 0   //状态码，0表示正常
}
```

####  【万事通】智能体状态同步接口
1. gpt-web模块提供一个callback接口给【万事通】，在【万事通】侧智能体被编辑时触发。一旦触发则gpt-web调用下面的【智能体列表查询】，获取当前智能体状态并对比更新已有配置的状态（根据appId），例如【智能体取消发布】和【智能体删除】  
2. 【万事通】需要提供一个【智能体列表查询】的接口供gpt-web模块调用，接口中需要返回智能体的状态【已发布】【取消发布】

```
POST /gpt/agent/sync

response body:
{
	"auditMsg": "", 
	"data": null,
	"messages": [],
	"statusCode": 0   //状态码，0表示正常
}
```
