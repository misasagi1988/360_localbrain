# 概念

- **规则**: sae规则+解析规则，假定sae规则配置是正确的，针对每一个sae规则的所有引用事件，评估哪些解析规则是满足该sae规则事件匹配条件的，哪些解析规则是不满足的，针对于不满足的解析规则，给出不满足的具体原因。
- **解析规则链**: 解析规则分为入口解析规则和非入口解析规则，非入口解析规则可以作为子规则，供入口解析规则使用，用于特定数据的匹配再解析。一个事件可以由同一个入口解析规则，使用多条解析规则链生成，不同的解析规则链输出的字段可能不一样，特定字段的值也不一样。
- **SIEM类型**: 360自有神经元，或三方数据
    sae规则的siem类型是通过规则配置来区分的；
    数据源评估，本质上是数据源引用的入口解析规则的评估，解析规则的siem类型通过引用它的数据源来区分，360自家的产品为xdr类型，三方数据源为siem类型。

# 评估设计思路

### 1. 评估准备工作

1. 获取系统数据源列表
2. 获取系统事件名称列表
3. 获取系统解析规则列表
4. 获取系统sae规则列表，分析规则引用事件，提取事件关键信息
5. 遍历事件名称，获取每个事件名称关联的所有解析规则链(每条链都包含该链可以解析出的字段名称，评估关注字段的所有可能值)，由于版本原因，较低版本解析规则链携带的信息可能不完整，需要通过分析解析规则链引用的所有解析规则进行补充。

### 2. SAE规则评估

SAE规则引用的事件可以分为以下几类：

| 事件类型 | 评估说明 |
| --- | --- |
| 系统定义事件 | 使用事件关联的所有解析规则链进行评估分析 |
| 全局事件 | 全局事件无法确定事件名称，要遍历所有系统事件，分析每个事件关联的所有解析规则链 |
| 匹配事件 | 同全局事件 |
| 内部事件 | 内部事件是sae规则触发生成的，需要分析能生成该内部事件的所有sae规则，通过他们的评估结果得到。对于内部事件，我们目前只关注触发规则是否有效，及内部事件过滤条件使用的字段触发规则是否全部输出，对于输出值，我们不做检查。 |
| 无效事件 | 事件不存在 |

规则评估处理流程参考：[[规则评估_重构流程.pdf]]

### 3. 数据源评估

数据源评估依赖的是sae规则评估的结果。
针对每一个数据源，分析sae规则评估的结果，得出其引用的入口解析规则，能够满足哪些sae规则的事件需求，哪些不满足，不满足的原因是什么，整理，得出结论文件。
在分析时注意，一个入口解析规则并不是所有sae规则都参与评估的，对于引用特定事件的sae规则，如果它无法生成相关事件，也要给出不满足的说明。
内部事件不参与数据源评估
数据源评估中，
适配规则含义：依赖当前的环境数据源配置，规则本身可以生效，而且数据源可以使规则中的一个事件满足条件而生效
未适配规则含义：依赖当前的环境数据源配置，规则本身无法生效，数据源参与了该规则其中一个或几个事件的分析，无法满足相关事件的过滤条件

# 评估异常捕获问题

异常范围：代码bug导致的异常
目前想到的异常类型：
- 分析事件filter语句，抛出异常，导致规则无法进行评估
- 评估事件过程中出现的异常
问题：
- 出现异常的规则，是否计入评估列表？
- 出现异常的规则可能会生成内部事件，导致连锁反应，引用该内部事件的规则无法进行评估。

想到的解决方法：
异常规则: 输出异常log，规则引用的其他事件也没有必要做评估，记录error message到该规则评估结果，该规则不录入评估结果列表(后台analysis在提取评估结果时滤除带error message的规则)；
关联内部事件连锁反应: 
- 事件只依赖一个规则，该规则异常: 引用内部事件的规则无法做评估，记录error message到该规则评估结果，该规则不录入评估结果列表(后台analysis在提取评估结果时滤除带error message的规则)；
- 事件依赖多个规则，有可以正常评估的规则: 使用其他正常评估的规则的评估结果做进一步评估
