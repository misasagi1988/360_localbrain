### 需求来源: 安全
安全同事有要求，对于威胁类事件，解析规则要输出threat_type，同时，需要有对应的普通模板sae规则，来匹配对应的event_name和threat_type，生成告警。
随着本脑在集成市面安全设备日志的解析支持日益增强，设备种类不断丰富，新产品的日志未能及时补充到SIM规范的威胁类型映射值，导致关联规则缺失。本文档旨在通过分析解析规则与SAE规则缺失比对，==找到因解析配置错误或缺乏对应的转换SAE规则而导致没有触发告警的分析结果==。
Comparison of missing analysis rules and SAE rules

### 设计思路:

#### 概述

提取相关的解析规则的事件名称、威胁类型字段，获取各个解析规则所能生成的事件名称、威胁类型的可能值。
针对每个解析规则解析出的事件名称、威胁类型的可能值，查看是否有对应的sae规则。sae规则的话，无需关注内置与否，启动与否，siem/xdr，只要存在即可。
```
{
    'page': 1,
    'page_size': 2000,
    'locale': 'zh_cn',
     'event_name': event_name,  # 使用去重后的事件名称，建议不用这个，event_name是模糊匹配的
     'random': '威胁类型 = ' + '\\\"' + threat_type + '\\\"',  # 使用去重后的威胁类型
     'silent': 0,
     'rule_type': 0,
     'status': 1,
     'rule_tag': '告警转换'
 }
```
基于规则评估，增加对比分析逻辑，添加相应代码。
新建分支: threat_rule_missing_analysis

#### 关键思路：

事件: 获取事件列表，过滤威胁类型非空的事件名称；这个可以用于过滤哪些解析规则需要被分析。
sae规则: 获取当前环境已经配置的普通模板类规则，用于缺失检查，按事件名称分类，找出各个事件名称关联的普通模板sae规则名称，filter。
解析规则: 获取解析规则列表，分析每个解析规则，得到其事件名称及威胁类型的配置信息；针对所有解析规则，获取其引用的子规则信息；
