### SIM对象调整

#### SIM对象调整：
 - 新增告警对象，保存告警的基本信息（类似“日志”对象）；
 - 修改现有威胁对象，增删必要的字段

#### 告警对象，主要描述告警基本信息，包含字段：
告警名称
关联分析规则名称
攻击场景
威胁可信度 (threat_confidence)，服务于“高置信/全面”模式
告警阶段
告警级别
ATT&CK Tactic ID
ATT&CK Technique ID
告警内容
告警处置建议
告警次数

#### 威胁对象，主要服务于研判目的，标为重要的属于威胁对象的核心字段，包含字段：
威胁名称
告警设备列表（重要）
攻击者列表（重要）
受害者列表（重要）
攻击结果（重要）
威胁识别特征（重要）
攻击载荷
IOC列表（重要）
失陷资产列表：支持IP、主机名称、主机标识（重要）
开始时间
结束时间
必选字段和重要可选字段可作为合并告警表头字段，且NDR、EDR类型都应使用此表头。

#### NOTE
告警必须包含"威胁"对象，威胁对象描述了告警的基础信息和用于研判的最重要信息。威胁对象在合并告警中的应用：
 - 威胁对象的"必选字段+重要可选字段"可作为告警列表的表头字段，其余可选字段可用于合并告警详情页的概览表格；
 - 威胁对象与其它实体的组合可描述不同类型的告警，如"威胁+主机 = 主机告警、威胁+邮件 = 邮件告警、威胁+流量=网络告警、威胁+流量+http协议=Web告警" ... 
 - 威胁对象所包含的信息理论上应能支持云端告警研判和数据积累

### 告警合并策略配置

#### 默认合并策略
SAE规则的场景选择决定该规则到底属于网络场景还是终端场景，不同的场景默认归并策略不同。默认合并策略如下：
 - 网络场景默认合并字段：规则id 源地址（代表攻击者）+目的地址（代表受害者）+告警名称+告警内容
 - 终端场景默认合并字段：规则id 主机IP（代表受害者）+告警名称+告警内容

#### 自定义合并策略
在“告警配置”勾选“自定义告警合并策略”开启自定义合并策略的配置。
**NOTE:**
 - 安全团队设计一些内置推荐的合并场景：1）这些场景只支持单选，选中后自动勾上下面的对应字段；2）允许用户额外再勾选其它字段，用户如果额外勾选后与推荐场景不符合，则去掉推荐场景上的勾选。
 - 合并字段支持多选。所有勾选字段的取值相同的告警才会被合并到一起。
 - 合并字段不能是数组类型（针对统计类规则，生成的告警某个字段有多值的情况，如目的地址组）。如果告警的输出字段是数组类型，所有数组类型的字段最多保存N个值。
 - 自定义合并字段必须是告警的输出字段，威胁对象的字段位于最上方，向下依次排列SAE规则中引用的其它对象字段。

### 告警输出字段

输出属性从上到下依次列出：
- 威胁对象的核心属性
- 附加的其它输出属性，可引用该告警事件源包含的所有对象实体的字段集合
- 如果输出属性为数组类型，**则保存关联日志中该字段出现过的所有值**，最多保存N个

### SAE规则配置页其它调整

- 移除“反转攻击角色”配置：攻击角色反转在新设计中可由“攻击者列表”、“受害者列表”的配置项所取代
- 增加“威胁可信度”配置：威胁可信度用于辅助研判，且未来可基于此字段提供“精准/全面”模式。
**若选择“使用日志的威胁可信度”，则将告警可信度设置为所有关联日志中最高的可信度取值；同理，合并告警的可信度设置为所有原始告警中最高的可信度取值。告警可信度是必须输出的，如果日志中不存在，输出为未知**

### SAE功能实现

#### 日志中threat_confidence求max

| 规则模板      | epl demo |
| ----------- | ----------- |
| normal      | `threat_confidence` as `threat_confidence`   |
| statistic   | max(`threat_confidence`) as `threat_confidence`  |
| FB/OR-FB   | {A.`threat_confidence`, B.`threat_confidence`, ...}.max() as `threat_confidence`   |
| RU   | {A.max(i -> `threat_confidence`), B.`threat_confidence`}.max() as `threat_confidence`    |
| NBF   | B.`threat_confidence` as `threat_confidence`        |
| NFB   | A.`threat_confidence` as `threat_confidence`        |
| any-order,not-occur   | 无法体现在epl中，自己在listener中实现    |

#### 提取日志中的所有字段值，输出为数组

| 规则模板      | epl demo |
| ----------- | ----------- |
| normal      | {`attr_A`} as `attr_B`   |
| statistic   | window(`attr_A`) as `attr_B`  |
| FB/OR-FB   | {A.`attr_A`, B.`attr_A`, ...} as `attr_B`   |
| RU   | A.selectFrom(i->`attr_A`).union({B.`attr_A`}) as `attr_B`    |
| NBF   | {B.`attr_A`} as `attr_B`        |
| NFB   | {A.`attr_A`} as `attr_B`        |
| any-order,not-occur   | 无法体现在epl中，自己在listener中实现    |

####攻击者列表、受害者列表

如果规则没有配置攻击者列表、受害者列表，规则自动输出，根据场景选择不同的字段。
EDR：攻击者列表为空，受害者列表优先取主机名称，如果没有，取主机ip
NDR/XDR: 攻击者列表为源地址组，受害者列表为目的地址组
攻击者类型、受害者类型信息，相关枚举值：
```
    HOST("主机"),
    IP("IP"),
    DOMAIN("域名"),
    FILE("文件"),
    OTHERS("其他");
```
  # 攻击者类型、受害者类型映射，类型分为五种，IP类型和其他类型没有列在上面
  # 只要配置的攻击者组、受害者组中引用的字段是以下field中的字段，则类型为对应的type
  # IP类型：如果配置的攻击者组、受害者组中引用的字段是IP或IP数组类型，则类型为IP
  # 其他类型：无法映射为以下几种，类型为其他
  mapper:
    - field: client_host_sign, server_host, host_guid, host_name
      type: 主机
    - field: domain_name
      type: 域名
    - field: file_sha1, file_md5, file_hash, file_hash_list, proc_md5, filename
      type: 文件
