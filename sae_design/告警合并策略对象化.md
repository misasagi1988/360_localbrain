# 告警合并策略对象化

标签（空格分隔）： SAE_DESIGN

---

## 合并策略数据表
```
CREATE TABLE `merge_strategy` (
  `id` varchar(20) NOT NULL COMMENT 'UUID主键',
  `name` varchar(255) NOT NULL COMMENT '策略名称',
  `description` varchar(1024) DEFAULT NULL COMMENT '策略描述',
  `alarm_name` varchar(255) DEFAULT NULL COMMENT '合并告警名称',
  `alarm_key` varchar(255) NOT NULL COMMENT '合并策略',
  `parsed_alarm_key` varchar(255) NOT NULL COMMENT '解析后的合并策略',
  `system` tinyint(1) NOT NULL DEFAULT '0' COMMENT '是否内置标记，0：用户自定义，1：系统内置',
  `logic_delete` tinyint(1) NOT NULL DEFAULT '0' COMMENT '策略是否被删除',
  `create_time` timestamp NOT NULL DEFAULT CURRENT_TIMESTAMP,
  `update_time` timestamp NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  UNIQUE KEY `name` (`name`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb3;
```

## 合并策略接口
- 查询接口
```
get: /api/cep/merge/strategy
查询参数:
page: 分页
page_size: 分页大小
name: 策略名称，模糊查询
ref: 引用规则名称，模糊查询
response body:
{
	"data": {
		"total": 3,   //总数
		"page": 1,   //当前页
		"list": [{
			"id": "131E7QH20000",   //合并策略id
			"name": "alarm-merge-strategy",   //合并策略名称
			"description": "desc",   //合并策略描述
			"alarmName": "",   //合并告警名称
			"alarmKey": "${关联分析规则名称}_${源地址}_${目的地址}_${告警名称}_${告警内容}",   //合并策略
			"system": true,   //是否内置
			"refRules": ["", ""]   //引用合并策略的规则名称
		},...]
	},
	"messages": [],
	"statusCode": 0   //状态码，0表示正常
}
```
