# SAE not occur 模板功能设计

标签（空格分隔）： SAE_DESIGN

---

### 目标与应用场景

设计目标：实现not occur模板，用于表达某特定事件在一段时间内没有发生，则生成告警。规则模板需支持配置分组。
应用场景举例：心跳检测

### Qradar实现思路

QRadar uses a watcher task that periodically queries the last time that an event was seen (last seen time), and stores this time for the
event, for each log source. The rule is triggered when the difference between this last seen time and the current time exceeds the number of seconds that is configured in the rule.
Qradar使用定时任务的方式，周期性的查询并存储接收到的最近一笔该事件的时间，当最近一笔该事件的时间与当前时间的时间差超过规则预设的时间窗口长度时，即触发规则生成告警。

### 设计思路探索

参考Qradar实现思路，结合esper引擎自身能力，及现有分析引擎的功能，整理了以下四种设计方案。
 - 方案1：
   记录last A的occur_time，记为a_occur_time，及当前日志时间的最大值max，记为max_occur_time。类似qradar，使用定时任务，周期性的检查max_occur_time与a_occur_time时间差是否超过设置的时间窗口长度，如果超过，则告警，定时任务执行周期可以配置为时间窗口长度，或者根据规则时间窗口长度进行更细粒度划分。   
   优点：不会有告警误报   
   缺点：需要使用timer task，检查逻辑在定时任务中实现，较为复杂；定时任务执行周期为时间窗口长度大小，极有可能会造成告警延迟；如果执行周期更细粒度，则每个规则都需要配置定时任务，schedule不一致，线程太多，代码逻辑复杂，可能也会影响sae的主要功能。
 - 方案2：
   同样，记录last A的occur_time，记为a_occur_time，及当前日志时间的最大值max，记为max_occur_time。但不使用定时任务，而是交由自定义函数来检查max_occur_time与a_occur_time时间差是否超过设置的时间窗口长度，如果超过，则告警。   
   优点：相比方案1更容易一些   
   缺点：需要额外增加判断逻辑，防止重复告警；事件驱动，如果环境没有日志过来，则无告警。
 - 方案3：
   esper的pattern语法，timer:interval
   优点：简单   
   缺点：没有采用日志时间，可能会因日志延迟导致告警误报 。  
 - 方案4：
   使用搜索告警来实现。 
   优点：简单   
   缺点：belong 信息组会有问题；搜索告警通过定时任务实现，同样存在告警延迟问题。

方案1和方案4使用定时任务实现，如果两个定时任务之间有1笔A数据过来，则需要再等到下一个定时任务执行时才可以检测到异常情况，如果时间窗口长度较大，告警延迟也会较大，这种情况是不允许的；方案3使用esper的系统时间，因日志延迟导致的告警误报问题也是不允许的。
方案2采用事件驱动，如果系统启动后一直没有事件过来，则无法触发告警，但客户现场一般不会存在这种状况；而且这种方案基于日志时间，实现起来较为简单，可满足告警实时触发的需求，综合考虑之后，选择方案2。

### 最终实现方案

 - 以规则为单位，记录每个规则的相关配置，包括时间窗口长度，分组字段，接收到日志的最大时间及最小时间。
 - 以分组字段值为单位，记录满足条件的A事件的最近一笔数据
 - 有日志过来时，检查该规则的A事件的所有分组：
   - 如果分组不存在，检查接收日志的最大最小时间差是否达到时间窗口长度，满足条件即生成告警；
   - 如果分组存在，遍历分组，检查每个分组中last A日志的时间与当前接收日志的最大时间差是否达到时间窗口长度，满足条件即生成告警。
   - 规则运行时，会记录接收日志的最小及最大时间，达到时间窗口长度后，如果lastA不存在，则强制更新接收日志的最小时间为当前接收日志的最大时间，防止下一笔数据过来时仍满足条件导致重复告警；如果lastA存在，则强制更新生成告警的last A的时间字段为接收日志的最大时间，防止下一笔数据过来时仍满足条件导致重复告警。
 - 自定义trigger listener，生成告警，鉴于模板表述的含义是事件未在指定时间窗口长度内生成，输出告警中无法使用事件的属性作为告警的属性。
