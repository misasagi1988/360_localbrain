# Vibe Coding Prompt 建议

对于全栈系统，建议采用分阶段递进式 prompt，而不是一股脑全塞进去。

请帮我从零开始设计一个「运维故障辅助处理系统」，并给出完整的技术方案。

  请：
  1. 所有技术选型都说明**为什么选它**（对比了哪些方案）
  2. 用我能看懂的方式解释，避免堆砌术语
  3. 推荐**最适合快速上手**的方案，而不是企业级最复杂的方案

  ---

  ## 系统要做什么

  帮助运维团队处理客户故障工单，核心能力：
  - **工单管理**：创建、分配、跟踪、关闭故障工单
  - **AI 智能分析**：接入大模型，自动分析故障原因、给出排查建议
  - **知识库**：从已解决工单自动沉淀处理经验，支持搜索和复用
  - **仪表盘**：一眼看到当前工单分布、处理进度

  目标用户：运维团队内部使用（5人规模）

  ---

  ## 请输出以下内容

  ### 一、技术栈选型

  对每一项给出**推荐方案 + 2-3个备选 + 选择理由**：
  - 前端框架
  - 后端框架（或全栈方案）
  - 数据库
  - UI 组件库
  - 大模型 API 接入方案
  - 部署方案（要考虑小白也能部署）

  ### 二、系统架构

  - 画出架构图（用文字/ASCII 即可）
  - 说明各模块之间怎么通信
  - 数据流向是怎样的

  ### 三、数据库设计

  列出所有需要的表，每张表包含：
  - 字段名、类型、说明
  - 表之间的关系（一对多、多对多等）

  ### 四、核心 API 设计

  列出所有需要的接口，每个接口说明：
  - URL、请求方法
  - 请求参数、返回数据
  - 业务逻辑简述

  ### 五、前端页面规划

  列出所有页面，每个页面说明：
  - 页面用途
  - 页面上有哪些组件/模块
  - 用户交互流程

  ### 六、开发步骤建议

  按优先级分阶段，给出：
  - 第一阶段做哪些（能跑起来的最小版本）
  - 后续逐步添加什么
  - 每个阶段预计多长时间

  ### 七、风险提示

  - 作为小白可能遇到哪些坑
  - 哪些部分建议找有经验的人帮忙
  - 有没有现成的开源项目可以参考或二次开发
