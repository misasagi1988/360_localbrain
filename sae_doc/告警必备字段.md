# 告警必备字段

标签（空格分隔）： SAE_DOC

---

告警必备字段整理：

| 字段       | 类型   | 描述     |
| ---------- | ------ | -------- |
| id | string | 告警id   |
| sae_rule_id | string | 规则id   |
| sae_rule_name | string | 规则名称   |
| alarm_name | string | 告警名称 |
| start_time | long | 告警开始时间 |
| end_time | long | 告警结束时间 |
| timpstamp | long | 告警写入es时间 |
| alarm_stage | int | 告警阶段，映射值：1-侦查/2-投放/3-利用/4-安装/5-控制/6-攻击 |
| alarm_level | int | 告警级别，映射值：0-提醒/1-警告/2-严重/3-紧急 |
| alarm_content | string | 告警内容 |
| alarm_advice | string | 告警处置建议 |
| threat_confidence | int | 威胁可信度，映射值：1-低/2-中/3-高 |
| rule_tags | string[] | 告警规则标签 |
| sim_entities | string[] | 告警规则引用实体 |
| ATTCK_Mapping | string | attck信息，多个以逗号分隔，格式tacId_techId |
| technique_id | string | technique id信息，多个以逗号分隔 |
| tactic_id | string | tactic id信息，多个以逗号分隔 |
| sim_name | string | 规则攻击场景名称 |
| sae_rule_type | string | 规则攻击场景名称 |
| alarm_type | int | 安全场景模型子模型，1-edr-process/2-edr-file/3-edr-registry/4-edr-network/5-ndr-network |
| scene_type | string | 安全场景模型，ndr/edr |
| merge_key | string | 告警合并策略 |
| direction_key | string | 事件合并策略 |
| attack_direction | int[] | 告警攻击方向，0-外对外/1-外对内/2-内对内/4-内对外，edr类场景不存在该字段 |
| sip_dip | object[] | 告警源地址目的地址统计合集，edr类场景不存在该字段 |
| event_id | string[] | 告警溯源日志id组 |
| attacker_type | string | 攻击者类型，主机/IP/域名/文件/其他 |
| victim_type | string | 受害者类型，主机/IP/域名/文件/其他 |
| attacker_array | string[] | 攻击者组 |
| victim_array | string[] | 受害者组 |
| src_address_array | string[] | 告警溯源日志源地址组 |
| dst_address_array | string[] | 告警溯源日志目的地址组 |
| data_source | string[] | 告警溯源日志数据源组 |
| data_source_array | string[] | 告警溯源日志数据源组 |
| client_host_sign_array | string[] | 告警溯源日志客户端标识组 |

告警要输出的日志的相关字段
源地址/目的地址 geo字段
源地址组、目的地址组的所有ip区分内外网，构造intranet数组和internet
ioc相关数组字段：related_hash_entity/related_domain_entity/related_intranet_ip_entity/related_internet_ip_entity/related_machine_entity


ICE安全事件感知引擎
通过预置的场景，将一些告警和日志总结再一起，试着去还原一个攻击的完整行为信息。
ice会给告警重新生成id，丰富化src_address、dst_address、occur_address的geo信息，根据内外网配置将其分别写入internet/intranet数组字段。ATTCK_Mapping/technique_id/tactic_id会被分解成数组。
